1. 디자인 원칙

현대적이고 세련된 미니멀리즘을 지향합니다. 불필요한 장식보다 정보 계층과 상호작용 상태를 명확히 표현합니다.

일관성: 토큰 중심 설계로 색상·간격·모서리·그림자·모션을 통일합니다.

명확성: 작업의 결과와 위험도를 명확히 알리는 카피와 시각적 신호를 사용합니다.

접근성: WCAG 2.1 AA 대비 기준, 포커스 가시성, 키보드 조작, 스크린리더 지원을 보장합니다.

3. 타이포그래피

시스템 산세리프 스택 사용. 본문 기본 16px, 줄간 1.6. 헤딩은 의미 중심으로 사용.

  • 헤딩: H1 28–36px, H2 24px, H3 20px
  • 본문: 16px
  • 보조텍스트: 14px, 대비 4.5:1 이상
  • 숫자/코드: 모노스페이스 사용
4. 레이아웃 · 간격 · 브레이크포인트

8px 베이스 그리드 사용. 공통 여백은 8·12·16·24·32·48px을 권장합니다.

  • 컨테이너: 최대 1200px, 좌우 패딩 24px
  • 브레이크포인트: sm 640, md 768, lg 1024, xl 1280
  • 카드: radius 16px, shadow-1/2, 내부 패딩 20–24px

6. 상태 · 피드백

입력·로딩·성공·경고·오류 상태를 색/문구/아이콘/애니메이션으로 일관되게 제공합니다.

  • 진행 중: 스켈레톤 또는 미니 스피너 사용. 2초 이상은 진행 정보 제공.
  • 성공: 토스트(3–5초 자동 닫힘) + 세부 작업은 화면 내 배지 상태 갱신.
  • 오류: 원인·해결 방법·재시도 버튼 제공. 파괴적 액션은 모달 확인.
저장 완료
네트워크가 불안정합니다
7. 경고/얼럿 가이드

얼럿 레벨과 사용

  • 정보(Info): 상황 공유. 작업 차단 없음. 예) 백그라운드 동기화 시작.
  • 성공(Success): 작업 완료. 예) 업로드 완료.
  • 경고(Warning): 위험 가능성 안내. 예) 저장되지 않은 변경사항.
  • 오류(Danger): 작업 실패/차단. 즉시 조치 필요. 예) 인증 만료.
동기화 진행 중
일부 기능이 순간적으로 느릴 수 있어요. 작업은 계속 가능합니다.
저장되지 않은 변경사항
페이지를 떠나면 변경사항이 사라집니다.
세션이 만료되었습니다
보안을 위해 다시 로그인해주세요.

카피 가이드

  • 구체적 원인: 무엇이, 왜 발생했는지 요약
  • 즉시 행동: 기본 버튼 하나, 보조 버튼 하나
  • 기술세부: 확장/접기 영역으로 분리(개발자용)

8. 엣지 케이스 & 사용 시나리오

1) 빈 상태(Empty State)

사용: 초기 데이터 없음, 필터 결과 없음.

  • 메시지: “아직 항목이 없습니다.”
  • 행동: 주요 생성 버튼 1개 제공
아직 항목이 없습니다.

2) 로딩 지연/타임아웃

  • 3초 초과: “데이터를 불러오는 중… 약간 시간이 걸리고 있어요.”
  • 10초 초과: 재시도 버튼+네트워크 도움말
데이터를 불러오는 중…

3) 네트워크 오류/오프라인

  • 메시지: “네트워크 연결을 확인해주세요.”
  • 행동: 재시도, 오프라인 임시저장(가능 시)
네트워크 연결을 확인해주세요.

4) 권한 부족

메시지: “이 작업을 수행할 권한이 없습니다.” 요청: 관리자

이 작업을 수행할 권한이 없습니다. 관리자에게 요청해주세요.

5) 파괴적 액션(삭제/초기화)

  • 확인 모달: 행동을 명시(예: “정말 삭제하시겠어요?”)
  • 부트스트랩: 확인은 위험색, 취소는 기본색
정말 삭제하시겠어요? 이 작업은 되돌릴 수 없습니다.

6) 저장되지 않은 변경사항 이탈

탐색 전 경고. “변경사항이 사라집니다. 지금 저장하시겠어요?” 행동: 저장 / 나가기

7) 파일 업로드 오류

  • 확인: 용량/형식/개수 제한을 구체적으로 표기
  • 메시지: “최대 100MB, mp4만 지원합니다.”

8) 중복/충돌(동시 편집)

낙관적 업데이트 실패 시 롤백+경고. “다른 사용자의 변경이 감지되어 병합이 필요합니다.” 비교 화면으로 안내.

9) 페이지네이션 경계

마지막 페이지에서 삭제 시 이전 페이지로 안전 이동. 빈 페이지 방지.

10) 시간대/포맷

표시는 로컬, 저장은 UTC. ISO 8601 권장. 예: 2025-10-31T14:05:00Z

11) 국제화(i18n) 길이 변화

버튼 최소폭 확보, 다줄 줄바꿈 허용. 아이콘-only에는 aria-label 제공.

12) 접근성 알림

  • 즉시 주의는 role="alert", 그 외 진행 정보는 aria-live="polite"
  • 포커스 트랩은 모달에만, 닫기 키(ESC) 제공
9. 접근성(A11y) · 국제화
  • 대비: 텍스트 대비 4.5:1 이상(대형 3:1)
  • 키보드: 모든 인터랙션 탭 가능, 포커스 링 제거 금지
  • 라이브 영역: 토스트/얼럿은 스크린리더에 전달
  • 언어 속성: <html lang="ko"> 유지
<div role="alert">세션이 만료되었습니다. 다시 로그인해주세요.</div>
10. 구현 권장 사항
  • 토큰 우선: 모든 색·간격·반경·그림자는 CSS 변수 의존
  • 상태 분리: 로딩/성공/경고/오류를 컴포넌트 Prop으로 통일
  • 접근성: 포커스, 역할(role), aria-* 일관화
  • 테마: html[data-theme] 스위치(라이트/다크)
/* 상태별 클래스 예시 */
.is-loading { opacity:.6; pointer-events:none }
.is-error { border-color: var(--danger-500) }
.is-success { border-color: var(--success-500) }
